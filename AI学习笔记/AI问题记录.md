## 📚 问题列表

### 1. Claude是什么？你怎么理解Claude这个东西？它跟cursor有什么不同？如果从传统使用来看，它类似于什么？

#### Claude 是什么
**Claude** 是 Anthropic 公司开发的大型语言模型系列（包括 Claude 3.5/4.5/4.6），而 **Claude Code** 是 Anthropic 官方的 CLI 工具，让你在终端中直接与 Claude 协作编程。

#### Claude vs Cursor

| 对比项 | Claude Code | Cursor |
|--------|-------------|--------|
| **定位** | 终端 AI 助手（CLI） | AI 代码编辑器（IDE） |
| **界面** | 纯命令行 | 类似 VS Code 的 GUI |
| **使用场景** | 终端操作、脚本自动化、代码审查 | 日常编码、项目开发 |
| **集成方式** | 可集成到任何终端 | 独立编辑器 |
| **优势** | 轻量、可脚本化、适合 DevOps | 可视化强、代码补全体验好 |

#### 传统类比
- **Claude Code** 类似于 **"超级版的 Git + ChatGPT"** —— 在终端中与你对话、执行命令、管理代码
- **Cursor** 类似于 **"集成了 AI 的 VS Code"** —— 写代码时自动补全、解释代码

### 2. Claude配置：包括快捷键设置、API配置、以及每个项目的独立配置是怎样的？（全局配置和项目特殊配置）

#### 📁 配置文件位置
```
全局配置: ~/.claude/settings.json
项目配置: .claude/settings.json
项目本地覆盖: .claude/settings.local.json
```

#### ⌨️ 快捷键配置
快捷键主要由**终端**控制，Claude Code 使用 readline：
- `Ctrl+R` - 历史搜索
- `Ctrl+L` - 清屏
- `Option+Enter` - 多行输入（macOS）
- 在 `~/.inputrc` 或终端设置中自定义

#### 🔑 API 配置（在 `settings.local.json`）
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your-api-key",
    "ANTHROPIC_BASE_URL": "https://api.anthropic.com"
  }
}
```

#### 🎯 项目独立配置
每个项目可以有独立的 `.claude/` 目录：
- `.claude/settings.json` - 项目特定权限和配置
- `.claude/CLAUDE.md` - 项目上下文和指南
- `.claude/skills/` - 项目自定义技能
- `.claude/rules/` - 项目规则

### 3. Claude有哪些实用技能，各个技能的作用是什么？

| 技能 | 作用 | 示例 |
|------|------|------|
| `/simplify` | 代码质量检查，优化改进 | "简化这段代码" |
| `/commit` | 自动生成 Git commit 信息 | `/commit` |
| `/loop` | 定时执行任务 | `/loop 5m /check-status` |
| `/plan` | 进入计划模式，安全分析代码 | `/plan` |
| `/fast` | 切换快速模式 | `/fast` |
| `/diff` | 查看未提交的更改 | `/diff` |
| `/memory` | 编辑项目记忆文件 | `/memory` |
| `/context` | 查看上下文使用情况 | `/context` |

### 4. Claude如何使用mode？

#### 模型切换
```bash
/model          # 切换默认模型
/model sonnet   # 使用 Sonnet 4.6
/model opus     # 使用 Opus 4.6（更强但更慢）
/model haiku    # 使用 Haiku 4.5（更快）
```

#### 使用场景
- **Sonnet** - 日常开发，平衡速度和质量
- **Opus** - 复杂任务、架构设计、深度分析
- **Haiku** - 快速查询、简单任务

#### Fast 模式
```bash
/fast on        # 启用快速模式（仍用 Sonnet，但输出更快）
/fast off       # 关闭快速模式
```

### 5. Claude学习计划怎样的？

#### 阶段 1：基础掌握（1-2周）
- ✅ 熟悉基本命令：`/help`, `/plan`, `/fast`
- ✅ 学会上下文管理：`/memory`, `/context`, `/compact`
- ✅ 掌握 Git 工作流：`/commit`, `/diff`
- ✅ 了解权限配置

#### 阶段 2：高效使用（2-4周）
- 🔝 学习计划模式：分析代码 → 制定计划 → 执行
- 🔝 掌握技能系统：自定义技能、使用 MCP 服务器
- 🔝 优化提示词技巧：精准描述需求
- 🔝 并行工作：使用 worktree 和多会话

#### 阶段 3：深度整合（持续）
- 🚀 项目集成：为每个项目配置 CLAUDE.md
- 🚀 自动化流程：Hooks、定时任务
- 🚀 团队协作：共享配置、代码审查
- 🚀 性能优化：成本控制、Token 管理

### 6. 现在面试经常会问到对AI的态度，以及你的AI实践，这种问题应该怎么回答？平时应该关注哪些东西有利于以后面试问及？

#### 💬 如何回答"你对 AI 的态度"

**模板回答**：
> "我认为 AI 是**增强工具而非替代品**。在我的开发工作中：
>
> 1. **效率提升**：用 Claude Code 进行代码审查、生成测试、快速原型，开发效率提升约 40%
> 2. **学习加速**：通过 AI 解释复杂代码、学习新技术栈，比如我用 Claude 深入学习了 SwiftUI 和异步编程
> 3. **质量把控**：AI 帮助发现潜在 bug 和优化点，但最终决策和代码质量由我负责
> 4. **持续实践**：我关注 AI 工具的局限性，知道何时该用、何时不该用
>
> 我认为未来的开发者是**"会使用 AI 的人"**，而不是被 AI 替代的人。"

#### 📚 平时应关注什么

| 方面 | 具体内容 |
|------|----------|
| **工具使用** | 熟练掌握 1-2 个 AI 工具（Claude/Cursor/GitHub Copilot） |
| **实际案例** | 记录 AI 帮你解决的具体问题、节省的时间 |
| **局限性认知** | 了解 AI 会犯错（幻觉、安全漏洞），保持批判性思维 |
| **行业动态** | 关注 AI 辅助编程的最新趋势（如 Agent、MCP） |
| **伦理考虑** | 数据安全、隐私保护、代码归属 |
| **团队实践** | 在团队中推广 AI 工具的最佳实践 |

#### 🎯 面试加分项

- **量化成果**："用 AI 自动化测试，覆盖率从 30% 提升到 80%"
- **具体案例**："用 Claude 帮助理解 10 年历史的遗留代码"
- **方法论**：总结出自己的一套 AI 工作流
- **教学能力**：教同事使用 AI 工具，形成团队规范
