## 5. 循环引用与解决方案

### 5.1 常见场景

#### 场景 1：delegate 循环引用

```objective-c
@interface ViewController ()
@property (nonatomic, strong) CustomView *customView;
@end

@implementation ViewController

- (void)setup {
    self.customView.delegate = self; // strong → strong，循环引用
}

@end

// ✅ 解决：delegate 使用 weak
@property (nonatomic, weak) id delegate;
```

#### 场景 2：Block 循环引用

```objective-c
// ❌ 循环引用
self.block = ^{
    NSLog(@"%@", self.name); // self → block → self
};

// ✅ 解决方案 1：__weak
__weak typeof(self) weakSelf = self;
self.block = ^{
    NSLog(@"%@", weakSelf.name);
};

// ✅ 解决方案 2：__strong dance（防止中途释放）
__weak typeof(self) weakSelf = self;
self.block = ^{
    __strong typeof(weakSelf) strongSelf = weakSelf;
    if (strongSelf) {
        NSLog(@"%@", strongSelf.name);
    }
};

// ✅ 解决方案 3：__block + 置空（仅 MRC）
__block ViewController *blockSelf = self;
self.block = ^{
    NSLog(@"%@", blockSelf.name);
    blockSelf = nil;
};
```

#### 场景 3：NSTimer 循环引用

```objective-c
// ❌ 循环引用：RunLoop → Timer → target(self)
@property (nonatomic, strong) NSTimer *timer;

- (void)setupTimer {
    self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                                  target:self
                                                selector:@selector(fire)
                                                userInfo:nil
                                                 repeats:YES];
}

// ✅ 解决方案 1：中间代理对象
@interface TimerProxy : NSProxy
@property (nonatomic, weak) id target;
@end

@implementation TimerProxy
- (NSMethodSignature *)methodSignatureForSelector:(SEL)sel {
    return [self.target methodSignatureForSelector:sel];
}
- (void)forwardInvocation:(NSInvocation *)invocation {
    if (self.target) {
        [invocation invokeWithTarget:self.target];
    }
}
@end

// 使用
TimerProxy *proxy = [TimerProxy alloc];
proxy.target = self;
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                              target:proxy
                                            selector:@selector(fire)
                                            userInfo:nil
                                             repeats:YES];

// ✅ 解决方案 2：Block-based Timer（iOS 10+）
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                             repeats:YES
                                               block:^(NSTimer *timer) {
    [self fire];
}]; // 注意：这里 block 仍持有 self，需要 __weak

// ✅ 最佳实践：结合 __weak
__weak typeof(self) weakSelf = self;
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                             repeats:YES
                                               block:^(NSTimer *timer) {
    [weakSelf fire];
}];
```

### 5.2 循环引用检测工具

#### FBRetainCycleDetector

**原理**：基于 DFS 深度优先遍历对象的强引用关系图，检测环。

**使用**：

```objective-c
FBRetainCycleDetector *detector = [FBRetainCycleDetector new];
[detector addCandidate:self];
NSSet *cycles = [detector findRetainCycles];
NSLog(@"Cycles: %@", cycles);
```

#### MLeaksFinder

**原理**：延迟检测 ViewController 是否释放（弹出 3 秒后检查）。

**使用**：

```objective-c
// 在 AppDelegate 中启动
- (BOOL)application:(UIApplication *)application
    didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    [MLeaksFinder startLeakTracking];
    return YES;
}
```

#### Xcode Memory Graph Debugger

**使用步骤**：

1. 运行 App
2. 点击 Xcode 底部的 "Debug Memory Graph" 按钮
3. 查看内存泄漏图标（紫色感叹号）
4. 查看引用链

## 7. 内存泄漏检测实战

### 7.1 Instruments Leaks

**使用步骤**：

1. Xcode → Product → Profile (⌘+I)
2. 选择 "Leaks" 模板
3. 运行 App，操作相关功能
4. 查看Leaks时间线，查看泄露详情
5. 点击泄露对象，查看引用链

### 7.2 Xcode Memory Graph

**使用步骤**：

1. 运行 App，暂停（⌘+⌃Y）
2. 点击底部 "Debug Memory Graph"
3. 查看内存泄露图标（紫色感叹号）
4. 查看右侧引用关系图

### 7.3 静态分析

**使用步骤**：

1. Xcode → Product → Analyze (⌘+⇧+B)
2. 查看分析结果
3. 修复警告
