# 图片优化-加载解码与降采样

> 一句话总结：**jpg/png 是压缩格式；真正占内存的是解码后的位图（Image Buffer）。优化=控制解码尺寸（降采样）+ 解码时机（子线程/预解码）+ 避免不必要的主线程峰值。**  
> 公屏/列表场景下与 **SDWebImage 等** 的配置思路，见 [面试模拟-03](../../09-面试复盘/面试模拟/面试模拟-03-消息列表卡顿优化.md) 与 [面试模拟-04](../../09-面试复盘/面试模拟/面试模拟-04-内存优化.md)。

---

## 1. 核心概念

### 1.1 加载 → 解码 → 渲染

1. **加载（Data Buffer）**：从 Asset / Bundle / 磁盘读出压缩数据（如 PNG、JPEG）。
2. **解码（→ Image Buffer）**：将压缩数据转为像素缓冲（RGBA 等），尺寸与像素数决定内存。
3. **渲染（→ Frame Buffer）**：交给 `UIImageView` / `CALayer` 显示。

**内存估算（RGBA 8bit）**：宽 × 高 × 4 字节。例如 1920×1080 解码后约 **7.9 MB**，若原图 jpg 仅数百 KB，说明「文件小 ≠ 内存小」。

### 1.2 `imageNamed:` 与 `imageWithContentsOfFile:`

- **`imageNamed:`**：有**系统级缓存**，首次退后台或内存警告时可能被清；适合重复 UI 小图。
- **`contentsOfFile`**：通常**无全局缓存**，更适合大图或一次性展示，避免常驻内存。

## 2. 底层原理

### 2.1 降采样（Downsampling）

当**展示区域远小于图片像素尺寸**时，应对 **ImageIO** 生成缩略图，只解码需要的像素，而不是全尺寸解码再缩放。

**Swift 示例（概念与旧笔记一致）**：

```swift
import ImageIO

func downsample(imageURL: URL, to pointSize: CGSize, scale: CGFloat) -> UIImage? {
    let maxDim = max(pointSize.width, pointSize.height) * scale
    let options: [CFString: Any] = [
        kCGImageSourceShouldCache: false
    ]
    guard let src = CGImageSourceCreateWithURL(imageURL as CFURL, options as CFDictionary) else {
        return nil
    }
    let downsampleOptions: [CFString: Any] = [
        kCGImageSourceCreateThumbnailFromImageAlways: true,
        kCGImageSourceShouldCacheImmediately: true,
        kCGImageSourceCreateThumbnailWithTransform: true,
        kCGImageSourceThumbnailMaxPixelSize: maxDim
    ]
    guard let cg = CGImageSourceCreateThumbnailAtIndex(src, 0, downsampleOptions as CFDictionary) else {
        return nil
    }
    return UIImage(cgImage: cg)
}
```

要点：`kCGImageSourceShouldCache: false` 避免缓存整图解码结果；`ThumbnailMaxPixelSize` 控住峰值。

### 2.2 异步解码与预解码

- 默认可能在**即将上屏**时主线程参与解码，易导致卡顿。
- 可在**串行队列**中把图片绘入 `CGBitmapContext` 预解码，再回主线程赋值（与 SDWebImage 思路一致：下载/读盘后尽快 decode）。

### 2.3 峰值与 `@autoreleasepool`

批量处理多图时在循环内套 `@autoreleasepool`，利于及时释放临时对象、压低峰值。

### 2.4 超大图

- 小控件显示大图：**降采样**。
- 微博/微信式「可拖动查看细节」：**`CATiledLayer`** 分块加载与绘制。

### 2.5 与离屏渲染的关系

圆角、阴影、`mask` 等若触发 GPU 离屏，列表快速滚动时 CPU/GPU 频繁切换上下文，既卡也费电。工程上常用 **CPU 预合成圆角**、`shadowPath`、控制 `shouldRasterize` 使用场景等（详见《UIKit-渲染管线与离屏渲染》）。

## 3. 关键问题 & 面试题

- **图片内存由什么决定？** 主要由**解码后位图尺寸与像素格式**，不是压缩文件体积。
- **从磁盘到上屏大致经过哪些缓冲？** Data Buffer → Image Buffer → Frame Buffer。
- **SDWebImage 为何感觉更顺滑？** 内存+磁盘二级缓存、下载后即异步解码、合并相同 URL 请求等（见《设计一个图片加载框架》）。

---

## 4. 实战应用

- 列表缩略图：**固定 cell 尺寸下的降采样** + 后台预解码。
- 与缓存框架配合：磁盘存 data，内存存 **decoded 的 UIImage** 还是只存 data，按内存策略权衡。
- iOS 10+：`UITableViewDataSourcePrefetching` / `UICollectionViewDataSourcePrefetching` 做预取。
