# 设计一个图片加载框架

> 一句话总结：

## 1. 需求分析

<!-- 建议涵盖：
  - 核心功能：URL → UIImage，展示在 UIImageView 上
  - 非功能需求：高性能、低内存、支持缓存、支持多种格式、线程安全
-->



## 2. 架构设计

<!-- 建议涵盖：
  - 整体架构分层：
    - API 层：UIImageView.setImage(url:)
    - 管理层：ImageManager（协调下载、缓存、解码）
    - 缓存层：MemoryCache（NSCache）+ DiskCache（FileManager）
    - 网络层：ImageDownloader（URLSession）
    - 解码层：ImageDecoder（后台线程解码 + 降采样）
    - 处理层：ImageProcessor（圆角、滤镜、缩放）
  - 数据流：请求 → 内存缓存命中？→ 磁盘缓存命中？→ 网络下载 → 解码 → 处理 → 缓存 → 回调
-->



## 3. 关键设计决策

<!-- 
  - 缓存策略：LRU、缓存 key 设计（URL hash）、内存/磁盘容量限制、过期策略
  - 优先级控制：可见 Cell 优先下载，快速滑动时取消不可见请求
  - 解码策略：后台线程解码、按目标尺寸降采样
  - 去重：相同 URL 的并发请求合并
  - 渐进式加载：Progressive JPEG
  - 占位图与错误图
  - 预取（Prefetching）机制
-->
