# 设计一个日志系统

> 一句话总结：

## 1. 需求分析

<!-- 建议涵盖：
  - 日志分级：Debug / Info / Warning / Error / Fatal
  - 高性能写入：不影响主线程、不丢日志
  - 支持上报：主动上报 + 远程回捞
  - 支持加密与压缩
-->



## 2. 架构设计

<!-- 建议涵盖：
  - 分层设计：
    - API 层：Logger.info("message") — 简洁易用
    - 格式化层：时间戳 + 线程 + 文件名:行号 + 日志级别 + 内容
    - 写入层：高性能本地写入
    - 上报层：网络上传（批量 + 压缩 + 加密）
  - 高性能写入方案：
    - mmap（内存映射文件）：写入性能接近内存，进程异常退出不丢数据
    - 对比方案：NSFileHandle / fwrite / NSOutputStream
  - 微信 Mars xlog 的设计思路
-->



## 3. 关键设计决策

<!-- 
  - mmap vs 直接文件 I/O 的 trade-off
  - 日志文件的切割策略（按时间 / 按大小）
  - 压缩算法选择（zlib / LZ4 / zstd）
  - 加密方案（AES）：先压缩再加密
  - 远程回捞机制：服务端下发指令 → 客户端上传指定时间段日志
  - 日志脱敏：敏感信息自动过滤
-->
