# 设计一个网络层中间件

> 一句话总结：

## 1. 需求分析

<!-- 建议涵盖：
  - 统一的网络请求入口
  - 支持拦截器链（类似 OkHttp Interceptor）
  - 支持重试、缓存、Mock、日志、认证等通用能力
  - 支持请求/响应的统一处理
-->



## 2. 架构设计

<!-- 建议涵盖：
  - 分层设计：
    - API 定义层：Protocol / Enum 描述接口（类似 Moya 的 TargetType）
    - 中间件层：Interceptor Chain（请求拦截 → 处理 → 响应拦截）
    - 传输层：URLSession / Network.framework
  - 拦截器设计：
    - AuthInterceptor：自动添加 Token、Token 刷新
    - LogInterceptor：请求/响应日志
    - RetryInterceptor：失败重试策略（指数退避）
    - CacheInterceptor：请求缓存
    - MockInterceptor：开发环境 Mock 数据
  - 责任链模式的实现
-->



## 3. 关键设计决策

<!-- 
  - 拦截器的执行顺序与优先级
  - 错误处理与统一错误码
  - 泛型响应解析（Codable）
  - 取消机制与生命周期绑定
  - 并发请求控制（最大并发数）
-->



## 4. 参考框架

<!-- 
  - Alamofire 的 RequestInterceptor
  - Moya 的 Plugin 机制
  - OkHttp 的 Interceptor Chain（跨平台思路）
-->
