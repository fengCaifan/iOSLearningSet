# 设计一个路由框架

> 一句话总结：

## 1. 需求分析

<!-- 建议涵盖：
  - 页面间解耦跳转
  - 支持 URL Scheme 外部跳转（DeepLink / Universal Link）
  - 支持参数传递与回调
  - 支持拦截（登录态检查、权限校验）
-->



## 2. 架构设计

<!-- 建议涵盖：
  - 核心组件：
    - Router：路由入口，负责 URL 匹配与分发
    - Registry：路由注册表（URL Pattern → Handler/ViewController）
    - Interceptor：路由拦截器链（登录、权限、降级）
    - Navigator：实际执行页面跳转（push/present/custom）
  - 注册方式：
    - 编译期注册：宏/注解（__attribute__((constructor))）
    - 运行时注册：+load / AppDelegate 统一注册
    - 约定式路由：基于类名规则自动匹配（Target-Action）
  - URL 匹配算法：Trie 树 / 正则
-->



## 3. 关键设计决策

<!-- 
  - 类型安全 vs 灵活性的 trade-off
  - 路由降级策略：路由未注册时的 fallback（H5 兜底页）
  - 参数传递方案：URL Query / Dictionary / 强类型 Model
  - 页面间回调的实现：Closure / Delegate / NotificationCenter
  - 多端路由统一（iOS/Android/H5 共用一套路由表）
-->
