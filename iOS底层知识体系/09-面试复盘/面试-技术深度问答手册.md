## 1. 技术面试问题

### 1.1 OC Runtime 相关

**问题 1：请解释 OC 的消息发送机制**

```
答案要点：
1. objc_msgSend(receiver, selector)
2. 消息发送流程：
   - 通过 isa 指针查找类对象
   - 在类对象的 method list 中查找方法实现（cache → method list）
   - 找到：跳转到方法实现（IMP）
   - 未找到：沿着继承链查找（父类 → 根父类 → NSObject）
3. 动态方法解析：resolveInstanceMethod / resolveClassMethod
4. 消息转发：forwardingTargetForSelector → methodSignatureForSelector → forwardInvocation
5. 如果仍处理不了：doesNotRecognizeSelector: 抛出异常

代码示例：
// 底层调用
objc_msgSend(obj, @selector(method:));

// 等价于
[obj method];
```

**问题 2：什么是 Method Swizzling？有什么应用场景？**

```
答案要点：
1. 原理：交换两个方法的 IMP（实现）
2. 实现：
   - 获取方法：class_getInstanceMethod
   - 交换方法：method_exchangeImplementations
3. 应用场景：
   - APM 监控（统计方法耗时）
   - 防止崩溃（数组越界、字典 nil key）
   - 调试日志
4. 注意事项：
   - 只能在 +load 中执行一次
   - 需要考虑类簇（NSArray 实际是 __NSArrayI）
   - 避免在 +load 中执行耗时操作
```

**问题 3：Category 的原理？为什么不能添加实例变量？**

```
答案要点：
1. 编译时：
   - Category 生成 .o 文件
   - 包含方法列表、协议列表、属性列表
2. 运行时：
   - Runtime 加载 Category
   - 将 Category 的方法合并到类对象的方法列表中
   - 方法追加到原方法列表前面（覆盖同名方法）
3. 不能添加实例变量的原因：
   - 实例变量的内存布局在编译时确定
   - 运行时无法扩展现有对象的内存
   - 解决方案：Associated Objects（关联对象）
4. 多个 Category 的加载顺序：
   - 取决于编译顺序
   - 后编译的 Category 方法会覆盖先编译的
```

**问题 4：KVO 的实现原理？**

```
答案要点：
1. KVO：Key-Value Observing，键值观察
2. 实现原理：
   - Runtime 动态创建 NSKVONotifying_ 子类
   - 修改观察对象的 isa 指针，指向子类
   - 子类重写 setter 方法：
     - willChangeValueForKey:
     - 调用父类的 setter（原始实现）
     - didChangeValueForKey:
     - 通知观察者
3. 手动触发 KVO：
   - 禁止自动触发：+automaticallyNotifiesObserversForKey: 返回 NO
   - 手动调用 willChangeValueForKey: / didChangeValueForKey:
```

**问题 5：KVC 的实现原理？**

```
答案要点：
1. KVC：Key-Value Coding，键值编码
2. setValue:forKey: 查找顺序：
   - setValue:forKey:（直接访问方法）
   - _set<Key>:（成员变量）
   - _<key>（成员变量）
   - <key>（成员变量）
   - 如果仍找不到：setValue:forUndefinedKey:（可重写，默认抛出异常）
3. valueForKey: 查找顺序：
   - get<Key>, <key>, is<Key>, _<key>（访问方法）
   - countOf<Key>, objectIn<Key>AtIndex:（集合访问）
   - _<key>, <key>（成员变量）
   - 如果仍找不到：valueForUndefinedKey:（可重写，默认抛出异常）
```

### 1.2 内存管理相关

**问题 6：ARC 的原理？weak 和 unsafe_unretained 的区别？**

```
答案要点：
1. ARC：编译器自动插入 retain/release/autorelease
2. weak vs unsafe_unretained：
   - weak：对象释放后自动置 nil，需要维护 weak_table
   - unsafe_unretained：对象释放后不置 nil（野指针）
3. weak 的实现：
   - objc_initWeak → objc_storeWeak
   - 注册到 weak_table（哈希表）
   - 对象 dealloc 时，weak_clear_no_lock 遍历置 nil
4. 应用场景：
   - delegate 使用 weak
   - Block 中捕获 self 使用 __weak
```

**问题 7：AutoreleasePool 的实现原理？**

```
答案要点：
1. 作用：延迟释放对象，避免内存峰值
2. 底层结构：
   - AutoreleasePoolPage 双向链表
   - 每页 4096 字节，存储对象指针
   - POOL_BOUNDARY（哨兵对象）标记边界
3. 与 RunLoop 的关系：
   - Entry：push
   - BeforeWaiting：pop + push
   - Exit：pop
4. 子线程：
   - 默认无 RunLoop，需要手动创建 @autoreleasepool
```

### 1.3 多线程与并发相关

**问题 9：GCD 和 NSOperationQueue 的区别？**

```
答案要点：
GCD：
- 底层 C 语言 API，性能高
- 队列：Serial、Concurrent
- 同步执行：dispatch_sync（可能死锁）
- 异步执行：dispatch_async
- 任务组：dispatch_group
- 信号量：dispatch_semaphore

NSOperationQueue：
- GCD 的面向对象封装
- 支持任务优先级
- 支持任务依赖（addDependency）
- 支持取消任务
- 可设置最大并发数

应用场景：
- 简单异步任务：GCD
- 复杂任务管理：NSOperationQueue
```

**问题 10：NSTimer 为什么会导致循环引用？如何解决？**

```
答案要点：
1. 原因：
   - Timer 强引用 target（self）
   - RunLoop 强引用 Timer
   - self 强引用 Timer
   - 形成循环引用：self → Timer → self

2. 解决方案：
   - 方案 1：中间代理对象（NSProxy）
   - 方案 2：Block-based Timer + weak self
   - 方案 3：手动 invalidate + 置 nil（MRC）

代码示例：
__weak typeof(self) weakSelf = self;
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                             repeats:YES
                                               block:^(NSTimer *timer) {
    [weakSelf doSomething];
}];
```


```swift
// 给定一个整数数组 nums 和一个整数 target，返回两数之和为 target 的索引
func twoSum(_ nums: [Int], _ target: Int) -> [Int] {
    var dict = [Int: Int]()

    for (i, num) in nums.enumerated() {
        let complement = target - num
        if let j = dict[complement] {
            return [j, i]
        }
        dict[num] = i
    }

    return []
}

// 复杂度分析：
// 时间：O(n)
// 空间：O(n)
```

```swift
// 编写一个函数来查找字符串数组中的最长公共前缀
func longestCommonPrefix(_ strs: [String]) -> String {
    guard !strs.isEmpty else { return "" }

    var prefix = strs[0]

    for str in strs[1...] {
        while !str.hasPrefix(prefix) {
            prefix = String(prefix.dropLast())
            if prefix.isEmpty {
                return ""
            }
        }
    }

    return prefix
}
```

**问题 3：无重复字符的最长子串（LeetCode 3）**

```swift
func lengthOfLongestSubstring(_ s: String) -> Int {
    var charIndex = [Character: Int]()
    var maxLength = 0
    var start = 0

    for (i, char) in s.enumerated() {
        if let index = charIndex[char], index >= start {
            start = index + 1
        }
        charIndex[char] = i
        maxLength = max(maxLength, i - start + 1)
    }

    return maxLength
}
```

```swift
func reverseList(_ head: ListNode?) -> ListNode? {
    var prev: ListNode? = nil
    var current = head

    while current != nil {
        let next = current?.next
        current?.next = prev
        prev = current
        current = next
    }

    return prev
}
```

```swift
func hasCycle(_ head: ListNode?) -> Bool {
    var slow = head
    var fast = head

    while fast != nil && fast?.next != nil {
        slow = slow?.next
        fast = fast?.next?.next

        if slow === fast {

        }
    }

    return false
}
```

**问题 7：二叉树的中序遍历（LeetCode 94）**

```swift
// 递归
func inorderTraversal(_ root: TreeNode?) -> [Int] {
    guard let root = root else { return [] }

    return inorderTraversal(root.left) + [root.val] + inorderTraversal(root.right)
}

// 迭代
func inorderTraversal(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    var stack = [TreeNode]()
    var current = root

    while current != nil || !stack.isEmpty {
        while current != nil {
            stack.append(current!)
            current = current?.left
        }

        current = stack.removeLast()
        result.append(current!.val)
        current = current?.right
    }

    return result
}
```

### 2.4 动态规划

**问题 8：爬楼梯（LeetCode 70）**

```swift
// 假设你正在爬楼梯。需要 n 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶
func climbStairs(_ n: Int) -> Int {
    if n <= 2 { return n }

    var dp = [Int](repeating: 0, count: n + 1)
    dp[1] = 1
    dp[2] = 2

    for i in 3...n {
        dp[i] = dp[i - 1] + dp[i - 2]
    }

    return dp[n]
}

// 空间优化
func climbStairs(_ n: Int) -> Int {
    if n <= 2 { return n }

    var prev1 = 1
    var prev2 = 2

    for _ in 3...n {
        let current = prev1 + prev2
        prev1 = prev2
        prev2 = current
    }

    return prev2
}
```

**问题 9：打家劫舍（LeetCode 198）**

```swift
// 给定一个代表每个房屋存放金额的非负整数数组，计算你不触动警报装置的情况下，一夜之内能够偷窃到的最高金额
func rob(_ nums: [Int]) -> Int {
    guard nums.count > 1 else { return nums.first ?? 0 }

    var dp = [Int](repeating: 0, count: nums.count)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])

    for i in 2..<nums.count {
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
    }

    return dp.last!
}

// 空间优化
func rob(_ nums: [Int]) -> Int {
    guard nums.count > 1 else { return nums.first ?? 0 }

    var prev1 = max(nums[0], nums[1])
    var prev2 = nums[0]

    for i in 2..<nums.count {
        let current = max(prev1, prev2 + nums[i])
        prev2 = prev1
        prev1 = current
    }

    return prev1
}
```

## 5. 面试技巧

### 5.1 准备阶段

**1. 技术准备**：
- 复习核心知识点（Runtime、内存管理、多线程等）
- 刷 LeetCode（数组、链表、树、动态规划）
- 了解最新的 iOS 技术（SwiftUI、Core ML）
- 准备项目经验（2-3 个代表性项目）

**2. 软技能准备**：
- 行为面专项：`行为面试/README.md` 与其中各篇（STAR + 追问预案 + 填空模板）
- 思考自己的优势和劣势（每条优势配一个**可复述事例**）
- 准备向面试官的问题（团队、技术栈、发版节奏）

**3. 材料准备**：
- 更新简历（突出项目经验和技术栈）
- 准备 GitHub 仓库（开源项目、博客）
- 准备 App Store 链接（个人 App）

### 5.2 面试中

**1. 自我介绍（2-3 分钟）**：
```
"你好，我叫 XXX，有 X 年 iOS 开发经验。

目前就职于 XXX 公司，负责 XXX App 的开发，主要使用的技术栈是 Swift、UIKit、Combine。

我的优势是：
1. 技术深度：对 Runtime、内存管理有深入研究，曾在公司内部分享过
2. 性能优化：主导过启动优化、卡顿优化项目，启动时间降低 50%
3. 团队协作：经常参与代码评审，指导初级工程师

最近在关注 SwiftUI 和 Core ML，并在个人项目中实践。

我对贵公司的 XXX 产品很感兴趣，希望能加入团队。"
```

**2. 回答技术问题的技巧**：
- 先说答案，再解释原理
- 结合代码示例
- 举实际应用场景
- 不确定时诚实说明

**3. 白板编程技巧**：
- 先沟通思路，再写代码
- 注意边界情况处理
- 写测试用例
- 讨论时间/空间复杂度

### 5.3 面试后

**1. 跟进**：
- 24 小时内发送感谢邮件
- 3-5 天后询问进度
- 保持专业态度

**2. 复盘**：
- 记录面试问题
- 总结不足之处
- 针对性改进

**3. Offer 谈判**：
- 了解市场行情
- 考虑综合因素（薪资、期权、工作内容、团队氛围）
- 不卑不亢，理性谈判
