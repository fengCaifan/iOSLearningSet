# 面试模拟 03：消息列表卡顿优化

> 对应整理题：**「列表卡顿怎么优化？」「消息瞬时暴增怎么不卡？」**  
> 技术主线：**线下/线上观测、RunLoop+信号量卡顿检测、MessageKit 式排版、手动布局、图片与气泡、Diffable 增量、流控**。

---

## Q1：语音房/IM 公屏卡顿，你怎么定位？

**答**  
- **线下**：Time Profiler + Core Animation + Allocations；**Mark** 前后对比；**压力回放**真实消息序列（富文本、图、气泡、icon），测 **FPS、主线程 CPU**。  
- **线上**：**CADisplayLink** 算 FPS；**RunLoop Observer + 子线程信号量**：在 `beforeSources` / `afterWaiting` 等阶段若 **超时** 认为卡顿，回调 **持续时长**（注意功耗与误报）。

---

## Q2：公屏业务上卡的主要原因？

**答**  
- **消息类型多、Cell 复杂**（气泡、图层、装扮 icon）。  
- **大房间瞬时消息多**，主线程 **布局 + 文本测量 + 解码** 叠加。  
- **AutoLayout 高频更新** 触发大量 `layoutSubviews` / 文本测量。

---

## Q3：优化总思路（一页版）？

**答**  
**超细粒度复用 + 稳定且增量的数据源更新**：减少 UI 重建、约束计算、临时对象分配，提高吞吐。

**对齐 MessageKit 思想**  
- 布局计算与视图展示 **解耦**。  
- **富文本提前在后台排版**（TextKit：`NSTextStorage` + `NSLayoutManager` + `NSTextContainer`），提前 **boundingRect / cell 高度**。  
- **多类型分样式**；layout **可缓存**。  
- **自动布局改手动 frame**：position 与 size **分开算**，降低高频抖动。

---

## Q4：图片与气泡怎么减 GPU/CPU？

**答**  
- **缩略图 / downsample**（服务端或客户端 decode 前缩放）；礼物等 **缓存缩略图**。  
- **禁止主线程解码**：后台出 `CGImage` 再主线程设图；SDWebImage 可配 `scaleDownLargeImages`、`avoidAutoSetImage`、`imageThumbnailPixelSize` 等（按实际栈答）。  
- **气泡**：`resizableImage`、layer-backed；静态复杂气泡可 **`shouldRasterize` 缓存**（说明 **动画/分辨率变化**下可能需权衡）。

---

## Q5：列表更新策略？

**答**  
- `UICollectionView` + **DiffableDataSource**。  
- **Snapshot增量 append**：从 `currentSnapshot` 只追加新消息，**小范围 diff**，避免整表刷新「布局地震」。  
- **消息上限**：如仅保留最近 **N条展示**（按产品）。  
- **流控**：缓冲队列 + **定时批量上屏**（例：每 0.5s 放行一批）；支持 **快进/暂停** 模式，既保流畅又可追历史。

---

## Q6：优化后如何验收？

**答**  
- 线下 FPS、卡顿次数；**AB**；必要时 monkey。  
- 指标用你司真实 **P95 帧耗时 / 卡顿率**（原整理中 FPS 约 50+ 仅作参考）。
