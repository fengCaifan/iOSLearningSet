# 面试模拟 04：内存优化

> 对应整理题：**「怎么做内存优化？」「OOM 怎么查？」**  
> 技术主线：**虚拟内存与 iOS 无 Swap、phys_footprint、Jetsam、工具链、大图与缓存、WKWebView、泄漏与循环引用、业务案例**。

---

## Q1：iOS 内存和桌面系统最大区别？

**答**  
- 使用 **虚拟内存**，但 **无 Swap 到外存**（不能把匿名页换到硬盘顶很久）。  
- 压力上来 → **内存警告** →仍不够 → **Jetsam** 按优先级杀进程。  
- 看 App 占用常用 **`phys_footprint`**（如 `task_vm_info`），比只看 `vm_size` 更贴近 **被杀风险**。

---

## Q2：Dirty / Compressed / Clean 怎么理解？

**答**  
- **Dirty**：已改写，系统回收代价高。  
- **Compressed**：不活跃 dirty 可被压缩；**解压瞬间** footprint 会冲高，也可能诱发尖峰 OOM。  
- **Clean**：可重建（如代码页、映射文件）。

---

## Q3：你怎么查内存问题？

**答**  
- **Memory Gauge**：适合 **瞬时尖峰**（Instruments 有时跟不上）。  
- **memgraph + vmmap / leaks / heap**。  
- **Allocations**：**Generation Mark** 对比两轮之间 **新增对象**（适合「滑动中不断创建」）。  
- **MLeaksFinder / FBRetainCycleDetector**：泄漏与环（各有局限，如 MLeaks 偏 UIView/VC）。

---

## Q4：图片内存怎么控？

**答**  
- 解码后约 **宽 × 高 × 4** 字节（近似）；应 **按显示尺寸 downsample**。  
- **`imageNamed` vs `imageWithContentsOfFile`**：小图频繁用前者；大图本地可用后者 **避免长期缓存**。  
- **远程大图**：慎无脑进内存缓存；SDWebImage 等配合 **降采样、解码线程**。

---

## Q5：WKWebView 白屏/内存？

**答**  
实现 `webViewWebContentProcessDidTerminate`，**reload** 恢复；进程被系统杀后内容进程可重建。

---

## Q6：业务侧做过什么？

**答（按你项目说）**  
- **独立图片缓存命名空间**（如语音房），限制 **memory/disk**，**退房/离开场景清理**。  
- Feed 刷新 **清旧缓存**，避免长期胀。  
- **案例**：scroll 回调里反复 `setImage` → 快速滑动内存涨；**标志位只执行一次**。  
- **案例**：`NSMutableAttributedString` + 某属性未设宽度导致 **巨大临时 rect** → Memory Gauge 抓尖峰，**注释二分**定位。

---

## Q7：循环引用常见？

**答**  
block 强 self、delegate 非 weak、Timer、通知未移除等；治理用工具 + Code Review。
